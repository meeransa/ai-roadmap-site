Plain English Decoder

JARGONTRANSLATED

Every AI term you'll encounter — explained as if you've never written a line of code. Bookmark this page. You'll refer back to it often in Months 1 and 2.
core AI terms
Token
AI's unit of text — roughly ¾ of a word. Every AI request costs tokens. More context = more tokens = more cost. As a PM, this is your main cost driver when building AI features.
Inferencing
The act of the AI "thinking" and producing an answer. Every time your product calls an AI, that's an inference. It costs money and takes time — both are product decisions you own.
Context Window
How much the AI can "read" at once — like a desk. Once full, something falls off. Bigger = better understanding but more expensive. You decide the right size for your use case.
LLM
Large Language Model. The AI brain behind tools like ChatGPT, Claude, and Copilot. You choose which LLM powers your agent based on cost, speed, and accuracy — like choosing an engine for a car.
System Prompt
The permanent instructions you give an AI before any user talks to it. Like an SOP or a job description. You write this as a PM — no code needed. This is your most powerful design tool.
Non-Deterministic
AI gives a slightly different answer every time. Unlike software where 2+2 always = 4. This changes how you write specs, how you test, and how you define "done."
RAG
Retrieval-Augmented Generation. The AI searches your own documents before answering — so responses are based on YOUR data, not just general training. Like giving AI a Toyota-specific handbook.
MCP
Model Context Protocol. The standard "plug" that lets AI connect to any external app or data source. Think USB-C for AI integrations — one standard that works everywhere. modelcontextprotocol.io
Agent
An AI that can plan, take actions, check its own work, and loop until a task is complete. Unlike a chatbot that just replies, an agent actively does things — search databases, send updates, trigger workflows.
Orchestration
How multiple AI agents are coordinated to work together. Like a conductor leading an orchestra — one agent plans, another executes, another checks quality. You design this flow as a PM.
Agent Sprawl
When every team builds their own agent without coordination — leading to duplicated effort, inconsistent data, and security risks. A real governance problem at Toyota scale that you prevent.
Agent Mesh
A smarter multi-agent model where agents hand off work to each other cleanly — like a relay race. Reduces sprawl and increases reliability.
Evals (Evaluations)
How you measure whether your AI is giving good answers. Think QA, but for AI outputs. As a PM, you define what "good" looks like — engineers build the automated tests.
Middleware Compression
Shrinking the information sent to an AI before it's processed — to reduce cost and speed up responses. Like summarizing a 100-page doc to key bullet points before asking AI a question about it.
Latency
How long it takes for AI to respond. 3 seconds is fine for a research tool; 0.5 seconds is required for a real-time customer-facing feature. You define the acceptable threshold for your product.
Vector Database
A special database that stores information so AI can search by meaning, not just keywords. "Car recall" and "vehicle defect" would match — even though the words are different.
